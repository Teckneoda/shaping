# Pay ranges

ℹ️ This has been looked into in the [Jobs Listing Fields Doc](https://www.notion.so/Jobs-Listing-Fields-For-Moving-Into-General-Classifieds-2bd2ac5cb2358085a57ada835369a06f?pvs=21).  Currently looking at adding the [following fields](https://www.notion.so/Jobs-Listing-Fields-For-Moving-Into-General-Classifieds-2bd2ac5cb2358085a57ada835369a06f?pvs=21) to General: `jobsPayRangeType`, `jobsPayFrom`, `jobsPayTo` (with values in cents).

This document describes available pay range values used on Jobs and their potential mapping on Classifieds schema.

## **Jobs Pay Range Structure**

| Field | Type | Purpose |
| --- | --- | --- |
| salaryFrom | integer | Minimum annual salary (e.g., 50000) |
| salaryTo | integer | Maximum annual salary (e.g., 75000) |
| hourlyFrom | integer | Minimum hourly rate (e.g., 15) |
| hourlyTo | integer | Maximum hourly rate (e.g., 25) |
| payRangeType | string | "salary" or "hourly" |

## **Classifieds Reusable Fields**

Classifieds has **generic spec fields** designed for category-specific data:

| Field | Type | Available |
| --- | --- | --- |
| specSubCatInt1 - specSubCatInt10 | integer | 10 fields |
| specSubCatFloat1 - specSubCatFloat2 | float | 2 fields |
| specSubCatString1 - specSubCatString10 | string | 10 fields |

We could store pay range using the category-specific fields:

### Option 1

| Jobs Field | Classifieds Field | Notes |
| --- | --- | --- |
| payRangeType | specSubCatString1 | "salary" or "hourly" |
| salaryFrom / hourlyFrom | specSubCatInt1 | Min value (dollars) |
| salaryTo / hourlyTo | specSubCatInt2 | Max value (dollars) |

### Option 2

| Jobs Field | Classifieds Field | Notes |
| --- | --- | --- |
| payRangeType | specSubCatString1 | "salary" or "hourly" |
| salaryFrom / hourlyFrom | specSubCatFloat1 | Min value (dollars) |
| salaryTo / hourlyTo | specSubCatFloat2 | Max value (dollars) |
